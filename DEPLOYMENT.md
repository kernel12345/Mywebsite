# 系统部署和使用文档

## 1. 系统概述

本系统是一个基于MySQL数据库的商品和文章管理系统，包含完整的后台管理功能。系统采用Vue 3 + Vite + Express + MySQL技术栈，具有以下功能：

- 用户认证和授权
- 商品管理（添加、编辑、删除、查询、批量操作）
- 文章管理（添加、编辑、删除、查询、批量操作）
- 分类管理
- 数据导入导出
- 安全措施（防SQL注入、XSS攻击等）
- 数据库查询性能优化和缓存机制

## 2. 环境要求

### 前端环境
- Node.js 14.0+ 
- npm 6.0+

### 后端环境
- Node.js 14.0+ 
- npm 6.0+
- MySQL 5.7+
- Redis 6.0+（用于缓存）

## 3. 部署步骤

### 3.1 前端部署

1. **安装依赖**
   ```bash
   cd frontend
   npm install
   ```

2. **构建生产版本**
   ```bash
   npm run build
   ```

3. **部署到静态文件服务器**
   将 `dist` 目录下的文件部署到 Nginx、Apache 等静态文件服务器。

### 3.2 后端部署

1. **安装依赖**
   ```bash
   cd backend
   npm install
   ```

2. **配置数据库连接**
   编辑 `backend/config/db.js` 文件，配置MySQL数据库连接信息：
   ```javascript
   const sequelize = new Sequelize('database', 'username', 'password', {
     host: 'localhost',
     dialect: 'mysql'
   });
   ```

3. **配置Redis连接**
   编辑 `backend/config/cache.js` 文件，配置Redis连接信息：
   ```javascript
   const redisClient = createClient({
     url: 'redis://localhost:6379'
   });
   ```

4. **启动后端服务**
   ```bash
   npm start
   ```

   或使用PM2进行进程管理：
   ```bash
   npm install -g pm2
   pm2 start app.js --name backend
   ```

## 4. 系统使用

### 4.1 管理员登录

1. 访问前端应用，点击登录按钮
2. 输入默认管理员账号：
   - 邮箱：admin@example.com
   - 密码：admin123
3. 登录后进入后台管理界面

### 4.2 商品管理

1. **添加商品**：点击"添加商品"按钮，填写商品信息并保存
2. **编辑商品**：点击商品列表中的"编辑"按钮，修改商品信息并保存
3. **删除商品**：点击商品列表中的"删除"按钮，确认后删除商品
4. **批量操作**：选择多个商品，进行批量删除或批量更新状态
5. **搜索和筛选**：使用搜索框和筛选条件查找商品
6. **导入导出**：使用"导入商品"和"导出商品"按钮进行数据导入导出

### 4.3 文章管理

1. **添加文章**：点击"添加文章"按钮，填写文章信息并保存
2. **编辑文章**：点击文章列表中的"编辑"按钮，修改文章信息并保存
3. **删除文章**：点击文章列表中的"删除"按钮，确认后删除文章
4. **批量操作**：选择多篇文章，进行批量删除或批量更新状态
5. **搜索和筛选**：使用搜索框和筛选条件查找文章
6. **导入导出**：使用"导入文章"和"导出文章"按钮进行数据导入导出

### 4.4 分类管理

1. **添加分类**：点击"添加分类"按钮，填写分类信息并保存
2. **编辑分类**：点击分类列表中的"编辑"按钮，修改分类信息并保存
3. **删除分类**：点击分类列表中的"删除"按钮，确认后删除分类
4. **批量操作**：选择多个分类，进行批量删除

## 5. 系统配置

### 5.1 前端配置

编辑 `src/services/api.js` 文件，配置后端API地址：
```javascript
const API_BASE_URL = 'http://localhost:3001/api';
```

### 5.2 后端配置

- **数据库配置**：`config/db.js`
- **Redis配置**：`config/cache.js`
- **安全配置**：`app.js` 中的helmet和rateLimit配置

## 6. 系统维护

### 6.1 数据库备份

定期备份MySQL数据库，确保数据安全。

### 6.2 日志管理

系统会在控制台输出错误日志，建议配置日志系统将日志保存到文件。

### 6.3 性能监控

使用Redis监控工具监控缓存使用情况，使用MySQL监控工具监控数据库性能。

## 7. 常见问题

### 7.1 数据库连接失败

- 检查MySQL服务是否运行
- 检查数据库连接配置是否正确
- 检查数据库用户权限是否正确

### 7.2 缓存不生效

- 检查Redis服务是否运行
- 检查Redis连接配置是否正确

### 7.3 前端无法访问后端API

- 检查后端服务是否运行
- 检查前端API地址配置是否正确
- 检查CORS配置是否正确
